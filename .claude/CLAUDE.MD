## Git-Workflow
- Nutze immer feature branches: git checkout -b feat/<name>
- Erstelle immer selbstständig Branches und Commits und nutze dafür fetch, push, pull usw.
- Commit messages auf Englisch, Conventional Commits Format
- Nach jedem abgeschlossenen Feature: git add -A && git commit
- Nach jeder abgeschlossenen Migration: commit mit git add -A && git commit -m "feat(db): <beschreibung>"
- Pushe nach jeder session deine änderungen.
- Immer bevor du einen neuen Branch erstellst, sollst du fetch und pull vom main machen
